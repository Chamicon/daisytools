1. Introduction

The Validator Iterator is a batch processing interface for the Daisy 2.02 Validator.

For general information on the Daisy 2.02 Validator, see the Daisy 2.02 Validator User Manual. This manual only describes features specific to the batch processing interface.

2. Using the Iterator

  1. Point to a "mother folder" (CTRL+O).

    A mother folder is typically a harddrive root or any other directory beneath which one or several DTBs reside. The Iterator will analyze every folder under the specified folder, and add all found DTBs to the batch process list.

  2. Set a destination folder for output reports (CTRL+P).

    The output reports will be sorted in automatically created subfolders named "fail" and "pass" beneath the folder you specify.

    The report filenames have the syntax dc:identifier_YYYYMMDDHHMMSS. If you validate a DTB at 12 o´clock on the first of March 2004, and its identifier is s1234, then the correpsnding report filename will be s1234_20040301120000. The naming convention is this messy and long to make sure no conflicts (duplicate filenames) occur.

    IMPORTANT! You can use the Iterator in a networked setting. For the Iterator to work as intended however, you need to map the network drive that you wish to work with. This is typically done using the "Tools/Map Network Drive" option in Windows Explorer.

    A successfully mapped networked drive is signified by that it appear with a "drive letter" (such as G:\ et cetera) on your local machine. Once this appears, you simply select the local drive name as mother folder or output report path.

    If you dont follow the instructions above, and instead specify folders by neighbourhood browsing, the iterator will likely not work.

  3. Locate all available DTBs (CTRL+L).

    Once the directory tree has been fully searched, the "dtb list" area of the user interface will be populated. Optionally, you can deselect certain of the DTBs in the batch process list by unchecking the checkbox adjacent to each DTB in the DTB List.

  4. Configure desired settings in the settings area.

    The Iterator supports modifying the light mode, disable audio tests and time flucuation settings. For more information on what these settings do, see the Daisy 2.02 Validator User Manual.

  5. Start the batch validation process (CTRL+R).

  6. Wait.

  7. Once the wait is over (the status log will say "done", and a system ping will (perhaps) be heard), go to the folder you specified and have a look at the results.
